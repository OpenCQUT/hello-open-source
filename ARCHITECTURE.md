# 架构说明

本项目采用 Docusaurus 作为静态文档站框架，目标是让内容维护者可以主要通过 Markdown / MDX 维护网站。

## 模块划分

```txt
src/pages/        独立页面：首页、资源雷达、路线图、贡献入口
src/data/         结构化数据：资源、路线、任务
docs/             主要文档内容
blog/             活动复盘、学习笔记、贡献者故事
static/           图片、图标、静态资源
.github/          Issue 模板、PR 模板、部署工作流
```

## 内容原则

- 首页负责引导；
- 文档负责解释；
- 资源页负责筛选；
- 路线图负责阶段规划；
- 贡献页负责把读者转化为贡献者。

## 视觉原则

使用现代科幻风，但不牺牲可读性：

- 深色优先；
- 高对比文字；
- 霓虹渐变作为强调，而不是正文背景；
- 卡片、边框、光晕用于分区；
- 动效只做轻量装饰。
