# 贡献指南

欢迎给 Hello Open Source 贡献内容。这个项目本身也应该成为新手练习开源贡献的入口。

## 你可以贡献什么

| 类型 | 示例 |
|---|---|
| 文档修正 | 错别字、链接失效、步骤不清楚 |
| 资源补充 | 公开课、书籍、工具、中文教程 |
| 学习路线 | 某一阶段的学习目标、任务和验收标准 |
| 活动材料 | Workshop 模板、学习小组计划、复盘文章 |
| 网站改进 | 样式、页面结构、组件、可访问性 |

## 贡献流程

1. Fork 仓库。
2. 新建分支。
3. 修改内容。
4. 本地运行并检查页面。
5. 运行构建，确认文档链接和页面编译通过。
6. 提交 Pull Request。

```bash
git checkout -b docs/add-course-guide
npm ci
npm run start
npm run build
```

## 内容质量要求

提交资源时，请至少说明：

- 资源适合谁；
- 前置知识是什么；
- 学完能做什么；
- 建议学习周期；
- 是否免费；
- 是否有中文材料；
- 为什么值得加入本站。

## 文风要求

- 少堆链接，多给路径；
- 少用抽象口号，多给可执行步骤；
- 解释术语，不默认读者已经知道；
- 优先中文说明，必要时补充英文原始链接；
- 不推荐未经验证的营销课程。

## PR 标题建议

```txt
docs: add CS50 study guide
fix: correct typo in first PR guide
feat: add resource radar page
style: refine sci-fi landing page
```

## Review 标准

维护者主要检查：

- 链接是否有效；
- 内容是否适合新手；
- 是否存在版权问题；
- 是否与现有结构重复；
- 是否能形成明确行动建议。
