# 项目实践

学开源最重要的是完成真实项目，而不是只看课程。

## 推荐实践项目

| 项目 | 目标 |
|---|---|
| 个人学习路线页 | 用 Markdown 写出自己的学习路线。 |
| 资源导读卡片 | 给公开课补充适合人群和学习建议。 |
| 第一次 PR 教程 | 把自己走过的步骤写成教程。 |
| 校园开源活动页 | 整理一次 Workshop 的流程和材料。 |
| 简单网页组件 | 给首页或资源页增加一个区块。 |

## 实践复盘模板

```md
# 项目名称

## 我做了什么

## 遇到的问题

## 我怎么解决

## 下一步可以改进什么
```

## 评价标准

一个好实践项目应该：

- 有明确目标；
- 有可展示产物；
- 有说明文档；
- 能被别人复用或改进。
