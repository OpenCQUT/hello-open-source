# 30 天开源入门计划

这个计划的目标不是学完所有知识，而是在 30 天内完成一次真实开源贡献。

## 第 1 周：认识和准备

| 天数  | 任务                               |
| ----- | ---------------------------------- |
| Day 1 | 阅读本站项目介绍和“开源是什么”。   |
| Day 2 | 注册 GitHub，了解仓库、Issue、PR。 |
| Day 3 | 安装 Git、Node.js、VS Code。       |
| Day 4 | 学习 Markdown 基础。               |
| Day 5 | Clone 本仓库并本地运行。           |
| Day 6 | 阅读贡献指南和 Issue 模板。        |
| Day 7 | 选择一个文档类任务。               |

## 第 2 周：工具链

- 学习 Git 基本命令；
- 完成一次本地 commit；
- 理解分支；
- 修改一个 Markdown 页面；
- 在自己的 Fork 中 push 分支。

## 第 3 周：第一次 PR

- 提交小范围 PR；
- 写清楚变更说明；
- 等待 Review；
- 根据反馈修改；
- 完成合并或总结失败原因。

## 第 4 周：复盘和扩展

- 写一篇贡献复盘；
- 给其他新手补充 FAQ；
- 推荐一个高质量资源；
- 尝试参与第二个 Issue。

## 最终产出

30 天结束时，你至少应该拥有：

- 一个 GitHub 账号；
- 一次 commit 记录；
- 一个 Pull Request；
- 一篇学习或贡献复盘；
- 对开源协作流程的基本理解。
