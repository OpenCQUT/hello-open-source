# Missing Semester 导读

Missing Semester 适合补齐工具链：Shell、Git、编辑器、调试、脚本和自动化。

## 这门课到底讲什么

Missing Semester 讲的是传统 CS 课程经常不系统讲、但真实项目每天都会用的工具：命令行、文本编辑器、Git、调试、构建、测试、远程机器、数据处理、安全和自动化。

## 它的目标

它的目标不是教一门编程语言，而是让你能在真实项目里：

- 找到文件、搜索代码、批量处理文本；
- 用 Git 保存历史、定位问题、协作修改；
- 阅读错误信息并用调试工具缩小范围；
- 用脚本、构建工具和 CI 减少重复操作；
- 更清楚地写文档、提 issue 和提交 PR。

## 每一类任务会带你做什么

| 主题 | 任务会带你做什么 | 对开源贡献的帮助 |
|---|---|---|
| Shell | 写管道、重定向、脚本和常用命令组合。 | 能本地运行项目、查日志、定位文件。 |
| Editors | 高效编辑、跳转、批量修改文本。 | 修改文档和代码时不被工具拖慢。 |
| Version Control | 练习提交、分支、历史、合并和定位问题。 | 完成 Fork → Branch → Commit → PR。 |
| Debugging | 读错误、用调试器、看日志、做性能观察。 | 遇到构建失败时能描述问题。 |
| Metaprogramming | 认识 Make、依赖、测试和 CI。 | 理解 PR 为什么需要自动检查。 |

## 为什么适合开源入门

开源贡献经常需要：

- 在命令行中运行项目；
- 使用 Git 管理分支；
- 阅读错误信息；
- 调整编辑器和环境；
- 写脚本自动化重复操作。

这些能力传统编程课不一定系统讲，但参与开源时非常实用。

## 推荐学习顺序

1. Shell；
2. Editors；
3. Version Control；
4. Debugging；
5. Metaprogramming。

## 推荐时长和难度

- 推荐时长：5–10 天完成核心 5 讲。
- 每讲投入：视频和笔记 1–2 小时，练习 1–3 小时。
- 难度：入门到中等。难点不在概念，而在是否真的敲命令、观察输出、修正错误。

## 验收任务

学完 Git 相关内容后，在本站完成：

```txt
Fork → Branch → Commit → Push → Pull Request
```

这条完整链路。
