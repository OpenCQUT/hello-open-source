# CS50 导读

CS50 是非常适合作为计算机科学入门的公开课，但它不是“轻松课”。

## 这门课到底讲什么

CS50x 2026 的主线从 Scratch 开始，然后进入 C、数组、算法、内存、数据结构，再切到 Python、SQL、HTML/CSS/JavaScript、Flask，最后以 Final Project 收束。它不是某个语言的速成课，而是用多种工具展示“如何把问题拆成程序、如何组织数据、如何构建一个小型应用”。

## 它的目标

这门课的目标是建立计算机科学入门框架：

- 理解程序控制流、数据表示和算法思维；
- 体验底层内存与数据结构带来的复杂度；
- 学会用更高层语言和数据库快速表达问题；
- 最后自己定义并完成一个项目。

它不适合用来快速学某个框架，也不适合只看视频不做题。

## 每一类任务会带你做什么

| 阶段 | 任务产出 | 训练重点 |
|---|---|---|
| Scratch | 做一个交互小作品 | 把想法拆成事件、条件、循环和状态。 |
| C 语言基础 | 写命令行小程序 | 输入输出、变量、数组、字符串、调试。 |
| 算法与内存 | 实现搜索、排序或内存相关程序 | 复杂度、指针、内存模型和错误定位。 |
| 数据结构 | 实现或使用链表、哈希表、树等结构 | 理解数据如何影响程序性能。 |
| Python / SQL | 用脚本和查询处理数据 | 用更少代码表达逻辑，理解数据持久化。 |
| Web / Flask | 做一个简单 Web 应用 | 页面、路由、数据库和用户交互。 |
| Final Project | 自己定义并完成作品 | 需求拆分、实现、展示和复盘。 |

## 适合谁

- 希望系统了解计算机科学的新手；
- 能接受英文材料或愿意查词的学习者；
- 愿意花数周时间做作业的人。

## 不适合谁

- 只想快速学会某个框架；
- 完全不愿意写作业；
- 只想看视频不实践。

## 建议学习方式

1. 先完成 Week 0–2，判断自己能否接受作业强度；
2. 每周至少完成一个 Problem Set，不要只看讲座；
3. 每个 Problem Set 结束后写 5–10 行中文复盘；
4. C、内存、数据结构阶段允许放慢，不要用“看懂视频”替代“能写出来”；
5. 最后把笔记、踩坑记录或资源导读贡献到本站。

## 推荐时长和难度

- 推荐时长：8–12 周；如果是零基础，可以拉长到 3–4 个月。
- 每周投入：6–12 小时，作业卡住时可能更久。
- 难度：中高。Week 0 友好，C、内存和数据结构部分是主要门槛。
