# 维护者手册

维护者的核心任务是降低参与门槛，同时保证内容质量。

## 每周维护任务

- 检查新 Issue；
- Review 新 PR；
- 清理失效链接；
- 补充 good first issue；
- 更新资源状态。

## Review 原则

- 对新手具体说明问题；
- 不要求一次做到完美；
- 把复杂建议拆成后续 Issue；
- 合并后感谢贡献者；
- 对不合适内容说明原因。

## 内容维护节奏

| 频率 | 任务 |
|---|---|
| 每周 | Review PR、处理 Issue。 |
| 每月 | 检查资源链接和活动材料。 |
| 每季度 | 调整学习路线和资源标准。 |
| 每年 | 回顾项目定位和社区目标。 |
