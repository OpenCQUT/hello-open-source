# 学习小组

学习小组适合持续推进公开课和开源实践。

## 推荐节奏

| 周期 | 内容 |
|---|---|
| 每周一次 | 分享学习进度和问题。 |
| 每两周一次 | 完成一个小任务或 PR。 |
| 每月一次 | 做一次公开复盘或展示。 |

## 组织方式

- 每组 4–8 人；
- 每次设一个明确目标；
- 每个人都要有小产出；
- 产出尽量提交到公开仓库；
- 活动结束后写复盘。

## 可选主题

- Missing Semester 共学；
- CS50 共学；
- 第一次 PR 小组；
- 文档冲刺；
- 校园开源项目共建。
