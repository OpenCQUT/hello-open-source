# 开源是什么

开源不是“把代码免费放到网上”这么简单。更准确地说，开源是一种公开协作模式：代码、文档、问题、讨论和改进过程都尽可能公开，让不同背景的人可以一起改进项目。

## 新手需要先理解的概念

| 概念 | 简明解释 |
|---|---|
| Repository | 项目仓库，通常包含代码、文档、配置和协作记录。 |
| README | 项目首页说明，告诉别人这个项目是什么、怎么用、如何参与。 |
| Issue | 问题、需求、任务或讨论。 |
| Pull Request | 请求把你的修改合并到目标项目中。 |
| Fork | 复制一份仓库到自己的账号下，方便修改和提交 PR。 |
| License | 许可证，说明别人可以如何使用、修改和分发项目。 |

## 开源贡献不只有写代码

以下都可以是有效贡献：

- 修正文档错别字；
- 给教程补充步骤；
- 维护资源链接；
- 翻译说明；
- 设计页面；
- 编写测试；
- 回答新手问题；
- 组织活动和复盘。

## 本项目推荐的开源入门顺序

1. 看懂仓库结构；
2. 学会 Git 和 GitHub 的基础操作；
3. 从文档类 Issue 开始；
4. 完成一次 PR；
5. 阅读 Review 反馈；
6. 再尝试资源整理、页面改进或代码贡献。
