# 许可证基础

许可证决定别人能否使用、修改、复制、分发你的项目。

## 新手需要知道什么

- 没有许可证，不代表别人可以随便使用；
- 代码许可证和内容许可证可以不同；
- 推荐在仓库根目录放 `LICENSE` 文件；
- 转载外部内容时要注意原始授权。

## 常见许可证

| 许可证 | 简单理解 |
|---|---|
| MIT | 宽松，允许使用、修改、分发，要求保留版权声明。 |
| Apache-2.0 | 宽松，包含专利授权条款。 |
| GPL | 传染性较强，衍生作品通常也需要开源。 |
| CC BY-SA | 常用于文档和内容，允许共享和改编，但要求署名和相同方式共享。 |

## 本项目建议

- 网站代码：MIT License；
- 文档内容：建议使用 CC BY-SA 4.0；
- 外部资源：只写导读和链接，不复制大段原文；
- 图片：优先使用自己制作的图或明确可用的开源素材。
