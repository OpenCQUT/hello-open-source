# 社区协作礼仪

开源协作不是只看技术水平，也看沟通质量。

## 提 Issue 时

应该包含：

- 你遇到的问题；
- 你期望发生什么；
- 你已经尝试过什么；
- 页面或代码位置；
- 截图或日志（如果有）。

不要只写：

```txt
这个有问题，快修。
```

## 提 PR 时

应该说明：

- 改了什么；
- 为什么改；
- 是否解决某个 Issue；
- 是否需要维护者特别检查某处。

## 收到 Review 时

- 先读懂反馈；
- 不确定就追问；
- 不把修改建议当成人身否定；
- 修改后回复说明已经处理。

## 做维护者时

- 给新手具体建议；
- 不只说“不行”；
- 把复杂任务拆小；
- 允许贡献者从文档和资源开始。
