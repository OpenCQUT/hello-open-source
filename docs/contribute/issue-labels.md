# Issue 标签体系

清晰的标签能帮助新手选择合适任务。

## 推荐标签

| 标签 | 含义 |
|---|---|
| `good first issue` | 适合第一次贡献。 |
| `documentation` | 文档相关。 |
| `resource` | 学习资源相关。 |
| `design` | 页面样式或视觉设计。 |
| `community` | 社区活动和运营。 |
| `needs-review` | 等待维护者审核。 |
| `help wanted` | 需要贡献者参与。 |

## 好的新手 Issue 应该怎样写

包含：

- 背景；
- 具体任务；
- 修改位置；
- 参考资料；
- 验收标准。

## 示例

```md
## 背景
CS50 页面缺少适合中文学习者的学习建议。

## 任务
在 docs/courses/cs50.md 中补充“常见困难”和“建议节奏”。

## 验收标准
- 文字清晰；
- 不复制外部大段原文；
- 给出具体行动建议。
```
