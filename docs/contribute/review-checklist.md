# Review 清单

维护者 Review 时可以使用以下清单。

## 文档类 PR

- 是否解决了一个明确问题；
- 是否适合目标读者；
- 是否有不必要的复杂术语；
- 链接是否有效；
- 是否存在版权风险；
- 是否保持了统一文风。

## 资源类 PR

- 是否说明适合人群；
- 是否说明前置知识；
- 是否说明学习周期；
- 是否说明为什么值得推荐；
- 是否与已有资源重复。

## 样式类 PR

- 是否影响移动端；
- 是否影响正文可读性；
- 是否与科幻风视觉系统一致；
- 是否增加了不必要的依赖。

## 给新手反馈的方式

差的反馈：

```txt
这个不行。
```

好的反馈：

```txt
这段建议保留，但需要补充适合人群。可以按“零基础 / 入门 / 进阶”拆成三句。
```
