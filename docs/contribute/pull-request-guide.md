# Pull Request 指南

PR 是一种协作请求：你请求维护者把你的修改合并进项目。

## PR 应该小而清晰

不建议第一次 PR 同时改：

- 首页样式；
- 多个文档页面；
- 配置文件；
- 资源数据；
- README。

一次只解决一个问题。

## PR 描述模板

```md
## 变更内容

## 解决的问题

## 截图或预览

## 检查清单
- [ ] 链接有效
- [ ] 本地运行正常
- [ ] 文案清晰
```

## 常见状态

| 状态 | 含义 |
|---|---|
| Open | PR 已提交，等待处理。 |
| Review requested | 等待维护者 Review。 |
| Changes requested | 需要修改。 |
| Approved | Review 通过。 |
| Merged | 已合并。 |
| Closed | 未合并并关闭。 |
