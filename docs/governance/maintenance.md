# 维护节奏

一个入门网站最容易出现的问题是：一开始热情很高，后续无人维护。

## 最小维护节奏

每周 30–60 分钟：

- 处理 Issue；
- Review PR；
- 检查资源链接；
- 新增 1–2 个新手任务。

## 每月维护

- 更新资源状态；
- 整理贡献者名单；
- 发布一次小结；
- 复盘学习路线是否清晰。

## 每季度维护

- 调整内容结构；
- 清理低质量资源；
- 评估新手任务完成率；
- 规划下一次活动。

## 维护者交接

维护者应该记录：

- 当前项目状态；
- 未完成任务；
- 资源审核标准；
- 活动模板；
- 常见问题。
