# Security Policy

本项目主要是文档网站。安全问题通常集中在：链接投毒、恶意脚本、供应链依赖和 GitHub Actions 权限。

## 报告范围

请报告以下问题：

- 恶意外链或伪装资源；
- 可执行脚本注入；
- GitHub Actions 配置风险；
- 依赖包安全问题；
- 可能泄露个人信息的内容。

## 不在范围内

- 普通错别字；
- 主观资源质量争议；
- 已经公开且无实际影响的信息。

## 处理流程

1. 提交 Issue，标题标注 `[security]`。
2. 维护者确认影响范围。
3. 修复后在变更记录中说明。

不要在公开 Issue 中粘贴敏感凭据、令牌或私人信息。
